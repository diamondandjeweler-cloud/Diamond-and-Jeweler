1. Executive Summary
BoLe is a dual‑sided recruitment platform that flips the traditional model:

Talents complete a system interview (20 questions + 20 preference ratings) and receive exactly 3 job offers at a time.

Hiring Managers (not HR) define role expectations and receive exactly 3 candidates at a time.

Hidden matching engine uses 16 years of proprietary feng shui / life‑chart data (DOB‑based) – weighted 30% – combined with behavioural tags (70%).

Market rate balancing educates both sides when expectations deviate.

“Fight the interview” active advocacy post‑match.

Progressive matching speed – 14 days initially, shortening to instant as database grows.

Target launch: Malaysia first, with a new dedicated domain (e.g., bole.my).

6. Life‑Chart Engine Tables (Phase 2)
These tables support your proprietary 16‑year‑old feng shui / DOB‑based matching. You will populate them with your data.

sql
-- Base life‑chart number by birth year range + gender
CREATE TABLE life_chart_base (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  start_date date NOT NULL,
  end_date date NOT NULL,
  gender text CHECK (gender IN ('male', 'female')),
  base_number integer NOT NULL,
  created_at timestamptz DEFAULT now()
);

-- Adjustments based on day and month (future use)
CREATE TABLE life_chart_adjustments (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  month integer CHECK (month BETWEEN 1 AND 12),
  day_from integer CHECK (day_from BETWEEN 1 AND 31),
  day_to integer CHECK (day_to BETWEEN 1 AND 31),
  gender text CHECK (gender IN ('male', 'female')),
  adjustment integer NOT NULL,
  description text,
  created_at timestamptz DEFAULT now()
);

-- Cache for computed scores (optional, for performance)
CREATE TABLE life_chart_cache (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  dob1 date NOT NULL,
  dob2 date NOT NULL,
  score numeric(5,2) NOT NULL,
  computed_at timestamptz DEFAULT now(),
  UNIQUE(dob1, dob2)
);
How it works:

Your algorithm takes hiring manager DOB + talent DOB.

Queries life_chart_base for each person’s base number using birth date range.

Applies adjustments from life_chart_adjustments (if any).

Produces a compatibility score (0–100).

That score is stored in matches.life_chart_score and used in final calculation:
Final = (Tag compatibility × 0.7) + (Life‑chart score × 0.3)

The actual scoring logic remains your proprietary secret.

7. Match State Machine
States and transitions (see full details in earlier spec).
Key states: generated → viewed → accepted_by_talent / declined_by_talent → invited_by_manager → hr_scheduling → interview_scheduled → interview_completed → offer_made → hired / offer_declined.
Expiry after 5 days of inactivity (configurable).
Automatic expiry cron job updates status to expired and triggers replacement matches.

10. Anti‑Ghosting & Expiry Logic
Expiry: 5 days of inactivity → status = expired.

Ghosting detection: If a user has 3 consecutive expired matches without any accept/decline, increment ghost_score. When score ≥ 3, temporarily reduce match priority (or pause matching) and show warning.

Replacement: On expiry, system generates a new match for that role/talent (respecting refresh limits) and logs in match_history.

11. Cold Start Strategy (Early Stage)
Manual seeding: Admin can select a role and pick talents (bypassing algorithm). Creates matches directly.

Fallback message: If role has < 3 eligible talents, show: “Not enough talents yet. We are growing our pool. You will be notified when we have 3 candidates. Estimated wait: 14 days.”

Auto switch: Once total talents > 500, disable manual seeding and rely on algorithm.

12. Role Versioning & Edit Handling
When a hiring manager edits an active role:

Insert old version into role_versions.

Increment roles.version.

Compare changes:

Minor (salary ±10%, same location/traits) → keep matches, recalc compatibility.

Major (title, experience_level, required_traits) → expire all current matches, regenerate new ones.

14. Multiple Talent Profiles
A talent can create several talent_profiles (e.g., “Marketing Manager”, “Data Analyst”). Each profile has its own interview answers, preference ratings, derived tags, and expected salary. Matches are linked to a specific talent_profile_id.

15. “Fight the Interview” SLA
Coach assignment within 1 hour of match acceptance.

First contact within 24 hours (tracked in coach_sessions.first_contact_at).

Dashboard countdown for talent.

If SLA missed → admin notification and coach reassignment.

16. Internal Matching Quality Metrics
Admin dashboard queries:

Match expiry rate = count(status='expired') / count(*)

Ghosting rate = users with ghost_score >= 3

Average time to first view = avg(viewed_at - created_at)

Refresh usage per role = sum of refreshed_by_manager from match_history

Interview success rate = count(hired) / count(interview_completed)

Feedback correlation = correlation between final_score and post‑interview feedback.

Copy and paste the entire script below into your new Supabase project’s SQL editor. Run it to create all tables, indexes, default configuration, and sample tags.

sql
-- ============================================================
-- BoLe Platform – Complete Database Schema
-- For Supabase (PostgreSQL)
-- Run this in a NEW Supabase project (not existing CRM)
-- ============================================================

-- Extensions
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS "pgcrypto";

-- Profiles (extends auth.users)
CREATE TABLE profiles (
  id uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  email text UNIQUE NOT NULL,
  full_name text NOT NULL,
  phone text,
  role text NOT NULL CHECK (role IN ('talent', 'hiring_manager', 'hr_admin', 'admin')),
  consents jsonb DEFAULT '{}',
  is_banned boolean DEFAULT false,
  ghost_score integer DEFAULT 0,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

-- Companies
CREATE TABLE companies (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name text NOT NULL,
  registration_number text UNIQUE NOT NULL,
  business_license_url text,
  website text,
  size text,
  industry text,
  primary_hr_email text NOT NULL,
  verified boolean DEFAULT false,
  created_at timestamptz DEFAULT now()
);

-- Hiring Managers
CREATE TABLE hiring_managers (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  profile_id uuid REFERENCES profiles(id) NOT NULL,
  company_id uuid REFERENCES companies(id) NOT NULL,
  job_title text NOT NULL,
  date_of_birth date NOT NULL,
  leadership_answers jsonb,
  leadership_tags jsonb,
  created_at timestamptz DEFAULT now()
);

-- Talents
CREATE TABLE talents (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  profile_id uuid REFERENCES profiles(id) NOT NULL,
  date_of_birth date NOT NULL,
  ic_url text,
  resume_url text,
  parsed_resume jsonb,
  privacy_mode text DEFAULT 'public' CHECK (privacy_mode IN ('public', 'anonymous', 'whitelist')),
  whitelist_companies uuid[] DEFAULT '{}',
  expected_salary_min integer,
  expected_salary_max integer,
  is_open_to_offers boolean DEFAULT true,
  created_at timestamptz DEFAULT now()
);

-- Talent Profiles (for multiple career tracks)
CREATE TABLE talent_profiles (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  talent_id uuid REFERENCES talents(id) ON DELETE CASCADE,
  profile_name text NOT NULL,
  interview_answers jsonb,
  preference_ratings jsonb,
  derived_tags jsonb,
  expected_salary_min integer,
  expected_salary_max integer,
  is_active boolean DEFAULT true,
  created_at timestamptz DEFAULT now()
);

-- Roles
CREATE TABLE roles (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  hiring_manager_id uuid REFERENCES hiring_managers(id) NOT NULL,
  title text NOT NULL,
  description text,
  department text,
  location text,
  work_arrangement text CHECK (work_arrangement IN ('remote', 'hybrid', 'onsite')),
  experience_level text CHECK (experience_level IN ('entry', 'junior', 'mid', 'senior', 'lead')),
  salary_min integer,
  salary_max integer,
  required_traits text[],
  status text DEFAULT 'draft' CHECK (status IN ('draft', 'active', 'paused', 'filled', 'expired')),
  version integer DEFAULT 1,
  market_rate_check jsonb,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

-- Role Versions
CREATE TABLE role_versions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  role_id uuid REFERENCES roles(id) ON DELETE CASCADE,
  version integer NOT NULL,
  snapshot jsonb NOT NULL,
  changed_by uuid REFERENCES profiles(id),
  created_at timestamptz DEFAULT now()
);

-- Matches
CREATE TABLE matches (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  role_id uuid REFERENCES roles(id) ON DELETE CASCADE,
  talent_profile_id uuid REFERENCES talent_profiles(id) ON DELETE CASCADE,
  compatibility_score numeric(5,2),
  tag_compatibility numeric(5,2),
  life_chart_score numeric(5,2),
  internal_reasoning jsonb,
  status text NOT NULL DEFAULT 'generated' CHECK (status IN (
    'generated', 'viewed', 'accepted_by_talent', 'declined_by_talent',
    'invited_by_manager', 'declined_by_manager', 'hr_scheduling',
    'interview_scheduled', 'interview_completed', 'offer_made',
    'offer_accepted', 'offer_declined', 'hired', 'expired', 'ghosted'
  )),
  viewed_at timestamptz,
  accepted_at timestamptz,
  invited_at timestamptz,
  expires_at timestamptz,
  refresh_count integer DEFAULT 0,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

-- Match History
CREATE TABLE match_history (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  role_id uuid REFERENCES roles(id),
  talent_profile_id uuid REFERENCES talent_profiles(id),
  action text CHECK (action IN ('generated', 'refreshed_by_manager', 'refreshed_by_talent', 'expired_auto', 'manual_admin')),
  previous_match_id uuid,
  created_at timestamptz DEFAULT now()
);

-- Interviews
CREATE TABLE interviews (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  match_id uuid REFERENCES matches(id) ON DELETE CASCADE,
  scheduled_at timestamptz,
  format text CHECK (format IN ('video', 'phone', 'in_person')),
  status text CHECK (status IN ('pending_hr', 'scheduled', 'confirmed', 'completed', 'cancelled', 'no_show')),
  coach_id uuid REFERENCES profiles(id),
  notes text,
  feedback_talent integer CHECK (feedback_talent BETWEEN 1 AND 5),
  feedback_manager integer CHECK (feedback_manager BETWEEN 1 AND 5),
  created_at timestamptz DEFAULT now()
);

-- Coaches
CREATE TABLE coaches (
  id uuid PRIMARY KEY REFERENCES profiles(id) ON DELETE CASCADE,
  specialization text[],
  max_concurrent_sessions integer DEFAULT 5
);

-- Coach Sessions
CREATE TABLE coach_sessions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  match_id uuid REFERENCES matches(id) ON DELETE CASCADE,
  coach_id uuid REFERENCES coaches(id),
  status text DEFAULT 'assigned' CHECK (status IN ('assigned', 'contacted', 'completed', 'missed_sla')),
  first_contact_at timestamptz,
  notes text,
  created_at timestamptz DEFAULT now()
);

-- Notifications
CREATE TABLE notifications (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES profiles(id) ON DELETE CASCADE,
  type text NOT NULL,
  channel text CHECK (channel IN ('email', 'in_app')),
  subject text,
  body text,
  data jsonb,
  read boolean DEFAULT false,
  sent_at timestamptz DEFAULT now()
);

-- Tag Dictionary
CREATE TABLE tag_dictionary (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tag_name text UNIQUE NOT NULL,
  category text CHECK (category IN ('boss_expectation', 'talent_expectation', 'behavioural')),
  weight_multiplier numeric(3,2) DEFAULT 1.0,
  is_active boolean DEFAULT true,
  created_at timestamptz DEFAULT now()
);

-- User Tags
CREATE TABLE user_tags (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES profiles(id) ON DELETE CASCADE,
  tag_name text REFERENCES tag_dictionary(tag_name),
  score numeric(4,3),
  source text,
  created_at timestamptz DEFAULT now()
);

-- Admin Actions
CREATE TABLE admin_actions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  admin_id uuid REFERENCES profiles(id),
  action_type text,
  target_type text,
  target_id uuid,
  old_value jsonb,
  new_value jsonb,
  reason text,
  created_at timestamptz DEFAULT now()
);

-- System Config
CREATE TABLE system_config (
  key text PRIMARY KEY,
  value jsonb NOT NULL,
  updated_at timestamptz DEFAULT now()
);

-- Market Rate Cache
CREATE TABLE market_rate_cache (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  job_title text,
  location text,
  experience_level text,
  min_salary integer,
  max_salary integer,
  median_salary integer,
  currency text DEFAULT 'RM',
  snapshot_date date DEFAULT CURRENT_DATE
);

-- Cold Start Queue
CREATE TABLE cold_start_queue (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  role_id uuid REFERENCES roles(id) ON DELETE CASCADE,
  talent_profile_ids uuid[],
  status text DEFAULT 'pending',
  created_at timestamptz DEFAULT now()
);

-- Subscriptions
CREATE TABLE subscriptions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  company_id uuid REFERENCES companies(id) ON DELETE CASCADE,
  plan text CHECK (plan IN ('free', 'pro', 'enterprise')),
  matches_per_month integer,
  refresh_limit integer,
  fight_the_interview_included boolean,
  price decimal,
